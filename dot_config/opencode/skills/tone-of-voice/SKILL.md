# Tone of voice

When drafting or revising any text Dan will send to others (including Slack posts, PR descriptions, issues, team emails, and chat replies), follow these style guidelines:

- **Conversational and flowing:** connect thoughts with commas into longer sentences; avoid short choppy ones. Comma-splices are fine and on-brand.
- **Plain prose:** no headers, bold, or bullets in casual/team writing. Just paragraphs. PRs can use structure only if genuinely needed, but prefer prose.
- **Matter-of-fact and low-drama:** state the problem and the plan directly. Avoid hedging filler; keep qualifiers light ("likely", "probably", "though that's unconfirmed").
- **Concrete and specific:** name exact identifiers inline (e.g. `enlighten_manager_token_production`, `e-auth-token`) and state behaviour tersely ("with the cookie returns 200, without returns 401").
- **Ownership plus a clear next step with a timeframe:** e.g. "I'll check on Monday and if so fix it asap".
- **First person, personal framing:** "my Home Assistant integration", "our client".
- **Slack-native when in chat:** lead with an emoji like `:information_source:`, sign off with `:slightly_smiling_face:`. Don't use Slack emoji in PRs/GitHub.
- **No em dashes:** use commas, colons, or parentheses instead.
- **British spelling:** behaviour, organise, etc.
